# 1. 연결 리스트 1
## 1.1. 추상 자료형(Abstract Data Type): ADT
구체적인 기능의 완성과정을 언급하지 않고, 순수하게 기능이 무엇인지를 나열한 것.

예) 지갑의 ADT
* 카드의 삽입
* 카드의 추출(카드를 빼냄)
* 동전의 삽입
* 동전의 추출(동전을 빼냄)
* 지폐의 삽입
* 지폐의 추출(지폐를 빼냄)

"그런데 기능의 나열이 어떻게 자료형임?"

만약 Wallet이라는 자료형을 '구조체'를 이용하여 정의하는 코드를 썻다고 해보자.

컴퓨터공학 측면에서, Wallet의 자료형을 구조체 정의만으로 완성된것은 아니다.
Wallet의 대한 연산의 종류를 결정하는것도, 자료형정의의 일부로 보아야한다.

만약 Wallet의 ADT를 정의한다면 다음과 같이 작성해야할 것이다.
```
* int TakeOutMoney (Wallet * pw, int coinNum, int billNum)
    - 첫 번째 인자로 전달된 주소의 지갑에서 돈을 꺼낸다.
    - 두 번째 인자로 꺼낼 동전의 수, 세 번째 인자로 꺼낼 지폐의 수를 전달한다.
    - 꺼내고자 하는 돈의 촙액이 반환된다. 그리고 그만큼 돈은 차감된다.

* void PutMoney (Wallet * pw, int coinNum, int billNum)
    - 첫 번째 인자로 전달된 주소의 지갑에 돈을 넣는다.
    - 두 번째 인자로 넣을 동전의 수, 세 번째 인자로 넣을 지폐의 수를 전달한다.
    - 넣은 만큼 동전과 지폐의 수가 증가한다.
```

"Wallet의 대한 정의도 넣어야 하는거 아님? 동전개수를 알고 싶을 수 도 있잖아."

그럴 경우엔 그냥 그러한 연산을 추가하는것이 더 좋다.
참고로 우리는 C의 stdio.h 파일을 뜯어보고 싶은 욕구가 들지 않았다. 파일을 뜯어보지 않아도 관련 연산을 처리할 수 있기 때문이다.

앞으로의 자료구조들은 ADT정의가 필수이다.
